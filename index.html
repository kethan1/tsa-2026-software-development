How it works

Your image is sent to the Claude API (claude-sonnet-4-20250514) which analyzes the visual content and generates a natural-language description optimized for screen readers. The description includes spatial layout, colors, text visible in the image, and contextual meaning — far beyond standard automated alt-text tools.

Why this matters

Approximately 8% of men and 0.5% of women have some form of color vision deficiency. By simulating their experience, designers can identify issues — like red/green status indicators — before shipping inaccessible products. This tool uses validated color transformation matrices to produce accurate simulations.

About AccessAI

🏆 TSA High School Software Development 2025

AccessAI was created for the Technology Student Association (TSA) 2025 National Conference Software Development event. The project addresses the theme of removing barriers and increasing accessibility for people with vision and hearing disabilities.

The platform runs entirely in-browser using the Web Speech API, Canvas API, and Claude AI — no installation, no account, no data stored. It was designed following WCAG 2.1 AA accessibility guidelines.

📋

Design process

Needs identified through interviews with assistive technology users. Prototyped using Figma; implemented iteratively with usability testing at each stage.

🛠️

Technologies

HTML5, CSS3, Vanilla JS, Web Speech API, Canvas API, Anthropic Claude API (claude-sonnet-4-20250514), Google Fonts.

♿

Accessibility features

Skip links, ARIA live regions, keyboard navigation, high-contrast mode, adjustable font sizes, screen reader labels throughout.

🌐

End users

Deaf and hard-of-hearing individuals, blind and low-vision users, accessibility designers, educators, and anyone needing captioning assistance.
